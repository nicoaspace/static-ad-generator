# Static Ad Generator (Claude Code + Nano Banana 2)

Generate 40+ production-ready static ad images for any brand — from brand research to finished creatives — entirely inside Claude Code.

## Overview

This skill replaces the manual Claude → Higgsfield workflow with a fully automated pipeline:
1. **Brand Research** → Claude builds a Brand DNA document via web search
2. **Prompt Generation** → Claude fills 40 template prompts with brand-specific details
3. **Image Generation** → Python script fires prompts to Nano Banana 2 via FAL API

No Higgsfield needed. No copy-pasting prompts. One command, 40+ ads.

---

## Prerequisites

- **Google AI Studio API key** set as environment variable: `export GOOGLE_API_KEY="your-key-here"`

## Phase 2: Prompt Generation

After the Brand DNA is complete, generate brand-specific prompts from the templates. Use the correct template file and JSON schema based on brand type.

### Prompt Generation — Product Brands

Take the 40 template prompts from the reference file and fill them in with detail specifically for [BRAND NAME] that aligns with the Brand DNA document, especially the Image Generation Prompt Modifier and the Ad Creative Style section.

For each template:
1. Replace all [BRACKETED PLACEHOLDERS] with brand-specific details
2. Prepend the Image Generation Prompt Modifier from the Brand DNA
3. Set the correct aspect_ratio based on the template (most are 1:1 or 4:5 or 9:16)
4. Determine if the template needs product reference images (needs_product_images: true/false)
   - Templates describing the actual product packaging = true
   - Templates that are pure lifestyle/UGC/editorial with no product shown = false
5. Include the product name and any specific product details provided

Output as a JSON file with this structure:
```json
{
  "brand": "Brand Name",
  "brand_type": "product",
  "product": "Specific Product Name",
  "generated_at": "ISO timestamp",
  "prompts": [
    {
      "template_number": 1,
      "template_name": "headline",
      "prompt": "Full completed prompt text ready for image generation...",
      "aspect_ratio": "4:5",
      "needs_product_images": true,
      "notes": "Any generation notes or copy that should be refined"
    }
  ]
}
```

Reference file for templates: See **references/template-prompts.md**

Save output as: ~/brands/{brand-name}/prompts.json

## Phase 3: Image Generation via Google AI Studio

Run the Python generation script to fire all prompts to Nano Banana 2 via the Gemini API. The `--type` flag selects product or service pipeline.

### How to Execute — Product brands

```bash
python skills/references/generate_ads.py --brand lmnt --type product
python skills/references/generate_ads.py --brand lmnt --type product --templates 1,7,13,15
python skills/references/generate_ads.py --brand lmnt --type product --dry-run
```

### How to Execute — Service / SaaS brands

```bash
python skills/references/generate_ads.py --brand notion --type service
python skills/references/generate_ads.py --brand notion --type service --templates 1,4,13
python skills/references/generate_ads.py --brand notion --type service --recommend   # pre-flight asset check
python skills/references/generate_ads.py --brand notion --type service --dry-run     # shows per-template eligibility
```

### Script Behavior

The generate_ads.py script (see references/generate_ads.py):

**Product pipeline** (`--type product`):
1. Reads prompts.json from the brand folder
2. Scans `product-images/` for all available reference images (flat folder)
3. For each prompt:
   - If `needs_product_images: true` → loads all product images with Pillow, calls `gemini-3.1-flash-image-preview` with inline image parts
   - If `needs_product_images: false` → calls the same model with text-only prompt
4. Saves generated images to `outputs/{template-number}-{template-name}/`
5. Saves the prompt text alongside each image as prompt.txt
6. Generates an index.html gallery showing all generated ads

**Service pipeline** (`--type service`):
1. Reads prompts.json from the brand folder
2. Scans `brand-assets/` subfolders (screenshots/, team/, logos/, icons/) for categorized assets
3. For each prompt:
   - Loads only the relevant asset categories based on `required_assets` + `preferred_assets` per template (smarter than product pipeline — not all images to all templates)
   - Calls `gemini-3.1-flash-image-preview` with the selected reference images
4. Same output structure as product pipeline

**`--recommend` command** (service only):
Pre-flight check that scans `brand-assets/` and reports which templates are eligible:
```
Brand assets scan:
  screenshots/    ✓  4 image(s)
  team/           ✗  empty
  logos/          ✓  3 image(s)
  icons/          ✗  empty

Template eligibility (35 total):
  ✓  READY   (28): S1, S2, S4, S5, ...
  ⚠  PARTIAL (4):  S3, S10, S20, S33 — would improve with: team/
  ✗  BLOCKED (3):  S19, S38, S8 — missing required: team/

Recommendation: Add team photos to brand-assets/team/ to unlock 7 more templates.
```

### Google AI Studio API Details

Model: `gemini-3.1-flash-image-preview`
Endpoint: `https://generativelanguage.googleapis.com/v1beta/models/gemini-3.1-flash-image-preview:generateContent`
Auth: `GOOGLE_API_KEY` environment variable

Python SDK (google-genai):
```python
from google import genai
from google.genai import types
from PIL import Image

client = genai.Client()  # reads GOOGLE_API_KEY automatically

# Text-to-image
response = client.models.generate_content(
    model="gemini-3.1-flash-image-preview",
    contents="Your full prompt here",
    config=types.GenerateContentConfig(
        response_modalities=["TEXT", "IMAGE"],
        image_config=types.ImageConfig(aspect_ratio="4:5", image_size="2K"),
    )
)

# Image editing with product reference images
response = client.models.generate_content(
    model="gemini-3.1-flash-image-preview",
    contents=["Your full prompt here", Image.open("product-front.png"), Image.open("product-angle.png")],
    config=types.GenerateContentConfig(
        response_modalities=["TEXT", "IMAGE"],
        image_config=types.ImageConfig(aspect_ratio="4:5", image_size="2K"),
    )
)

# Extract image from response
for part in response.candidates[0].content.parts:
    if part.inline_data:
        with open("output.png", "wb") as f:
            f.write(part.inline_data.data)
```

Aspect ratios: 1:1, 4:5, 5:4, 3:4, 4:3, 2:3, 3:2, 9:16, 16:9, 21:9
Resolution options: 512 (0.5K), 1K, 2K, 4K (default: 2K)
Reference image limits: up to 10 product/object images, up to 4 character images per call

## Key Technical Notes

- **Product brands**: Product images are critical for quality. The more reference images you provide (front, back, angled, lifestyle), the better the model matches the real product in output. 1-3 images is the sweet spot. Drop PNGs/JPGs in `product-images/` and the script handles encoding automatically.
- **Service brands**: Screenshot quality matters hugely. Use clean browser screenshots (no browser chrome), Figma exports, or official press kit assets — not photos-of-screens. Organize into `brand-assets/` subfolders: `screenshots/`, `team/`, `logos/`, `icons/`. Each template receives only its relevant asset categories (smarter than product pipeline). Run `--recommend` before generating to verify coverage.
- Reference images are passed inline (base64 via Pillow) — no pre-upload to external storage needed.
- Aspect ratios matter for ad placement. 1:1 for feed, 4:5 for feed (more real estate), 9:16 for stories/reels. The templates specify the correct ratio.
- Resolution: Default is 2K for production-quality output. Use 1K for faster test runs, 4K for hero assets.
- Images per prompt: 4 images generated per prompt by default, giving you options to pick the best output. A full 40-template run = 160 images.
- Cost (Google AI Studio): ~$0.067/image at 1K, ~$0.10/image at 2K, ~$0.151/image at 4K. A full 40-template run at 4 images each at 2K = ~$16. Use --templates to generate selectively and control cost. Free tier available for testing (rate-limited).
- Copy refinement: The generated copy from Phase 2 is functional but generic. For best results, mine customer reviews (via Apify scraping) and inject real customer language before running Phase 3.
